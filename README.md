# Portfolio Website

A modern, responsive portfolio website built with HTML, CSS, and JavaScript. Each section is organized as a separate page for better structure and maintainability.

## Features

- **Multi-Page Structure**: Each section is a separate HTML page for easy organization
- **2x2 Grid Layout**: Projects displayed in a clean 2x2 grid with square boxes and borders
- **Responsive Design**: Works seamlessly on desktop, tablet, and mobile devices
- **Interactive Elements**: Animated project cards and navigation
- **Modern UI**: Clean and professional design with minimal styling
- **Project Detail Pages**: Each project has its own detailed page

## Structure

```
portfolio/
├── index.html      # Home/Work page (2x2 grid of projects)
├── about.html      # About me page
├── resume.html     # Resume page
├── contact.html    # Contact page
├── project-1.html # Project 1 detail page
├── project-2.html  # Project 2 detail page
├── project-3.html  # Project 3 detail page
├── project-4.html  # Project 4 detail page
├── styles.css      # All CSS styles (shared across pages)
├── script.js       # JavaScript functionality (shared across pages)
└── README.md       # This file
```

## Pages

1. **Work (index.html)**: 2x2 grid showcasing 4 projects with square boxes and borders
2. **About (about.html)**: Personal information
3. **Resume (resume.html)**: Professional experience and skills
4. **Contact (contact.html)**: Contact information and social links
5. **Project Pages**: Individual detail pages for each project

## Customization

### Personal Information
- **Home Page**: Update your name in `index.html` (line ~16)
- **About Page**: Modify content in `about.html`
- **Projects**: Update project titles and links in `index.html`
- **Resume Page**: Update experience and skills in `resume.html`
- **Contact Page**: Update contact information in `contact.html`

### Projects
- Update project titles in `index.html` (work-item-title)
- Modify project detail pages (project-1.html through project-4.html)
- Add project images by replacing the placeholder divs

## Usage

1. Open `index.html` in a web browser
2. No build process required - works directly in the browser
3. For production, consider:
   - Adding a backend for the contact form
   - Optimizing images
   - Minifying CSS and JavaScript
   - Setting up a hosting service
